# SaaS Script-Driven Landing Page System

This system allows any static HTML website to dynamically load and render landing pages controlled from a centralized backend.

## 🚀 Client-Side Implementation

Clients only need to add a single script to their HTML `<head>` and configure their server to redirect all traffic to `index.html`.

### 1. Script Embed
Add this script to the `<head>` of your website:

```html
<script src="https://your-backend-domain.com/tracker.js" async></script>
```

### 2. Server Configuration (Fallback Routing)

**Apache (.htaccess):**
```apache
RewriteEngine On
RewriteCond %{REQUEST_FILENAME} !-f
RewriteRule ^ index.html [L]
```

**Nginx:**
```nginx
location / {
    try_files $uri /index.html;
}
```

## 🛠️ System Architecture

### 1. Tracker Script (`tracker.js`)
Located at `backend/public/tracker.js`. It performs the following:
- Detects the current `hostname` and `pathname`.
- Fetches the landing page content from `GET /api/page`.
- Replaces the entire document using `document.write()`.
- Automatically captures all form submissions and sends them to `POST /api/leads`.
- Supports Google Analytics (`gtag`) for `page_view` and `conversion` events.

### 2. Backend APIs

#### `GET /api/page`
**Query Params:**
- `domain`: The client's hostname (e.g., `client-site.com`).
- `path`: The current URL path (e.g., `/seo-offer`).

**Response:**
```json
{
  "status": "success",
  "html": "<!DOCTYPE html>...",
  "projectId": "...",
  "pageId": "..."
}
```

#### `POST /api/leads`
**Body:**
```json
{
  "name": "John Doe",
  "email": "john@example.com",
  "phone": "1234567890",
  "domain": "client-site.com",
  "pageUrl": "https://client-site.com/seo-offer",
  "path": "/seo-offer"
}
```

**Response:**
```json
{
  "success": true,
  "thankYouUrl": "https://client-site.com/thank-you",
  "message": "Lead captured successfully"
}
```

## 🗄️ Database Schema (MongoDB)

### Pages (`Page` model)
Stores the landing page content and mapping.
- `slug`: The URI path mapping (e.g., `seo-offer`).
- `domain`: (Optional) Restrict page to a specific domain.
- `landingPageContent`: Full HTML/CSS generated by AI.
- `thankYouUrl`: Redirect destination after lead submission.

### Leads (`Lead` model)
Stores lead data from dynamic pages.
- `name`, `email`, `phone`, `message`
- `domain`: The client domain where the lead was captured.
- `url`: The full URL of the landing page.
- `pageSlug`: Reference to the page slug.
- `projectId`: Association with the parent project.

---

## 🎯 Fail-Safe Behavior
- If the API fails or no page is found for the path, the script will do nothing, allowing the original `index.html` content to remain visible.
- Form submissions include a fallback check; if the lead capture fails, it logs the error but prevents the site from breaking.
